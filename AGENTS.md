# AGENTS.md

Working agreement for AI agents (and humans) modifying this codebase.

## Project summary

`office-converter` is a small HTTP service that converts spreadsheet files
between **XLSB**, **XLSX**, and **ODS** using **LibreOffice** (`soffice`) in
headless mode. It is a **Go 1.26+** application that ships as a single
**static, UPX-compressed** binary built on `debian:12-slim`. It is the Go
sibling of the Python/FastAPI and PHP/Slim implementations in the parent
workspace.

The repo name is `office-converter`; the Docker image, the binary name, the
Cobra command tree, and the OpenAPI title all match.

## Runtime model

- **Binary**: `office-converter serve` (Cobra). Static, CGO disabled, UPX-
  compressed by `make build` and by the release workflow. Link-time
  metadata (`Version`, `BuildDate`, `GitCommit`) is injected via
  `-ldflags -X office-converter/cmd.{Version,BuildDate,GitCommit}=â€¦`.
- **HTTP framework**: **Echo v5** (`github.com/labstack/echo/v5`).
  `Serve(cfg Config) error` is the only public entry point from
  `cmd/serve.go`; everything else is package-private.
- **Conversion engine**: `soffice --headless` invoked via
  `exec.CommandContext`, wrapped in `context.WithTimeout` so the
  per-conversion deadline is enforced by Go's context machinery
  (cancellation SIGKILLs the process). Each call gets its own isolated
  `-env:UserInstallation=file://â€¦/lo-profile-<rand>` directory via
  `os.MkdirTemp("", "lo-profile-")`, cleaned with `defer os.RemoveAll`.
- **Concurrency cap**: `OFFICE_MAX_CONCURRENT_CONVERSIONS` (default 2),
  enforced by a process-local **buffered channel** of `struct{}` used as
  a counting semaphore (`internal/server/convert.go`).
  - `select { case sofficeSemaphore <- struct{}{}: ...; case <-ctx.Done(): return }` to acquire.
  - The request context is the **only** cancellation path: if all slots
    are busy, the handler blocks until the client disconnects or the
    server is shut down. There is no wall-clock slot-timeout
    (the Python sibling uses 30 s; the PHP sibling uses 30 s) â€” adding
    one is fine, but the pattern must stay request-context aware.
- **Upload size**: enforced by `middleware.BodyLimit(maxUploadBytes)`
  (Echo's built-in), which returns **413** on overflow.
- **Temp dirs**: every request creates `<prefix>-<rand>/` via
  `os.MkdirTemp`. The four prefixes mirror the routes â€”
  `convert-req-`, `xlsb-to-xlsx-req-`, `xlsx-to-ods-req-`, `ods-to-xlsx-req-`
  â€” and are cleaned with `defer os.RemoveAll(workDir)` (no `BackgroundTasks`
  / `shutdown_function` indirection like the other ports).
- **Logging**: `log/slog` (stdlib, Go 1.26). Echo's
  `middleware.RequestLoggerWithConfig` produces one structured log line
  per request; `slog.Error` is used inside handlers on conversion
  failure.
- **Graceful shutdown**: `signal.NotifyContext` on `SIGINT` and
  `SIGTERM`, with `server.graceful_timeout` (default 15 s) handed to
  Echo's `StartConfig.GracefulTimeout`.
- **TLS**: built-in. `--tls` + `--tls-cert` + `--tls-key`, or the
  `[tls]` block in `config.toml`, or the matching `OFFICE_TLS_*` env
  vars. Cert and key are required when `tls.enabled = true` â€” `Serve`
  returns an error and the process exits non-zero otherwise.

## HTTP surface

| Method | Path                            | Description                                  |
|--------|---------------------------------|----------------------------------------------|
| GET    | `/`                             | Drag-and-drop web UI (HTML, no JS deps)      |
| GET    | `/healthz`                      | Liveness probe â†’ `200 ok` (text/plain)       |
| HEAD   | `/healthz`                      | Same probe, body-less                        |
| POST   | `/api/v1/convert`               | Smart endpoint (auto-detect direction)       |
| POST   | `/api/v1/convert/xlsb-to-xlsx`  | `.xlsb` â†’ `.xlsx` (typed)                    |
| POST   | `/api/v1/convert/xlsx-to-ods`   | `.xlsx` â†’ `.ods` (typed)                     |
| POST   | `/api/v1/convert/ods-to-xlsx`   | `.ods`  â†’ `.xlsx` (typed)                    |
| GET    | `/api/v1/openapi.json`          | OpenAPI 3 spec (only when `swagger.enabled = true`, else 302 â†’ `/`) |
| GET    | `/docs`                         | Swagger UI HTML (same gating)                |

Every `POST /api/v1/convert*` URL accepts **two** content types:

- `multipart/form-data` with field `file=<binary>` â†’ binary download.
- `application/json` with body `{"file": "<base64>", "filename": "..."}`
  â†’ JSON envelope (always).

You can also force a JSON envelope from a multipart upload with
`?format=json` or `Accept: application/json`.

### Auto-detect map (smart endpoint)

```
xlsb â†’ xlsx
xlsx â†’ ods
ods  â†’ xlsx
```

Anything else returns **HTTP 415**.

### HTTP status codes

| Code | When |
|------|------|
| 200  | Successful conversion (binary or JSON body) |
| 400  | Missing `file` field, invalid JSON, invalid base64, missing `filename` on the smart JSON endpoint |
| 413  | Upload body exceeds `--max-upload-size` (Echo `BodyLimit`) |
| 415  | Extension mismatch on typed endpoint, or unsupported source on smart endpoint |
| 422  | LibreOffice conversion failed (corrupt / unsupported file / timeout) |
| 500  | Internal error (workspace prep, file read, etc.) |
| 502  | Misconfiguration (TLS without cert/key) â€” bubbles out of `Serve` as a Go error |

The custom `HTTPErrorHandler` in `server.go` returns `{"error": "..."}`
for paths under `/api/` or when the client sent `Accept: application/json`,
and plain text otherwise.

## Coding conventions

- **Go 1.26+ features welcome**: `log/slog`, `signal.NotifyContext`,
  `os.MkdirTemp`, `errors.Is`, etc. Match the existing code.
- **Module layout**: `package main` lives only in `main.go`. `cmd/*` is
  its own package (so `-X office-converter/cmd.Version=â€¦` works at
  link time). All application logic lives under `internal/server` and
  must stay unexported. The OpenAPI types live under `internal/api` and
  are generated.
- **One file, one job**:
  - `server.go` â€” Echo bootstrap, middleware, error handler, TLS.
  - `convert.go` â€” subprocess + concurrency + temp cleanup.
  - `handlers.go` â€” HTTP handlers + JSON/multipart pipelines.
  - `routes.go` â€” URL â†’ handler wiring.
  - `config.go` â€” `Config` struct + mapstructure tags.
  - `templates.go` â€” inline HTML/Swagger strings.
- **Generated code**: `internal/api/api.gen.go` is produced by
  `oapi-codegen` from `api/openapi.yaml` + `api/oapi-codegen.yaml`. Do
  not hand-edit. Regenerate with `make generate` (requires
  `oapi-codegen` on `PATH`; the `tools/tools.go` blank import pins the
  version in `go.mod`).
- **Errors**: handlers return `error`. Validation failures use
  `c.JSON(status, ErrorResponse{Error: msg})` via the
  `jsonError(c, status, msg)` helper in `handlers.go`. Do not write
  directly to `c.Response()` and do not call `c.String` from a JSON
  route â€” keep the 404 / 500 / `HTTPErrorHandler` paths in `server.go`
  as the single source of truth for non-API error shaping.
- **Context plumbing**: every `exec.CommandContext` and every blocking
  operation that can be cancelled must take `c.Request().Context()` (or
  a child of it). The semaphore acquire uses `select { â€¦ case
  <-ctx.Done(): return ctx.Err() }` so a disconnected client unblocks
  the slot acquisition immediately.
- **Logging**: use `slog` (`slog.Info`, `slog.Error`, `slog.LogAttrs`).
  Do not use the deprecated `log` package or `fmt.Println` in
  production code paths.
- **No comments unless asked** â€” types and identifiers carry the
  meaning; the codebase has minimal inline narration. Short doc-comments
  on exported symbols are kept where they earn their place
  (e.g. `ConvertTo` in `convert.go`).
- **Static binary**: `CGO_ENABLED=0` and `go build -trimpath
  -ldflags="-s -w"`. The Makefile adds `-X` flags for the build
  metadata in `cmd/version.go`; keep those three names in sync
  (`Version`, `BuildDate`, `GitCommit`).
- **No new third-party deps** without a deliberate change to
  `go.mod` / `go.sum`. The current direct dependencies are intentionally
  few: `echo/v5`, `cobra`, `viper`, `oapi-codegen/{v2,runtime}`,
  `kin-openapi`.

## Concurrency contract â€” read before changing

`OFFICE_MAX_CONCURRENT_CONVERSIONS` is enforced by a **buffered channel
of `struct{}`** acting as a counting semaphore. The slot is released
via `defer func() { <-sofficeSemaphore }()` immediately after the
function returns, so the cap is exact and process-local.

The consequences:

- The cap is **per binary instance**. Running multiple binaries (or
  multiple containers) gives `N Ă— cap` total slots. This is the
  intended behaviour for a stateless HTTP service.
- Acquisition is `select`-based and **only the request context can
  cancel it**. There is no independent wall-clock timeout for "waiting
  for a slot" â€” a stalled request that holds a slot will block other
  requests until the client disconnects or the server shuts down. The
  Python / PHP siblings both implement a 30 s slot-wait timeout. If
  you add one here, keep the cancellation path context-aware so client
  disconnects still unblock the goroutine.
- `initConverter` must be called once at startup, before any handler
  runs; `Serve` does this. Do not lazy-init the channel from a handler
  â€” it would race on first request.
- Do not replace the buffered channel with a `golang.org/x/sync/Semaphore`
  or a mutex unless you also preserve the `context.Context`
  cancellation semantics.

## OpenAPI / code generation

- `api/openapi.yaml` is the **single source of truth** for the HTTP
  surface, request/response shapes, and the JSON `ConvertRequest` /
  `ConvertResponse` field names.
- `api/oapi-codegen.yaml` is configured for `models: true`,
  `client: true`, `embedded-spec: true`. The output lands in
  `internal/api/api.gen.go`.
- `tools/tools.go` is the standard Go 1.24+ pattern for tracking
  tooling in `go.mod` (the `//go:build tools` constraint keeps it out
  of the production binary).
- When you change a route, status code, or schema, edit the YAML and
  run `make generate`. Commit both files in the same commit.
- The handlers in `internal/server/handlers.go` use **local Go
  structs** (`convertRequest`, `convertResponse`, `ErrorResponse`)
  rather than the generated `api.*` types â€” the Echo v5 idioms and
  field names there match the YAML exactly, so there is no need to
  import the generated package at the handler layer. (`api.GetSpecJSON`
  is the one place that does use the generated package, to serve the
  embedded spec at `/api/v1/openapi.json` when `swagger.enabled = true`.)

## Build, run, test

Local (requires `soffice` in `PATH` and Go 1.26+):

```bash
make build            # optimised static binary + UPX
make build-debug      # binary with debug symbols, no UPX
make run              # runs bin/office-converter serve
# or:
go run . serve --port 9000
```

Docker:

```bash
make docker-build
PORT=8080 make docker-run
```

Integration tests (the authoritative suite â€” there are no Go unit
tests):

```bash
make generate-samples           # one-time; needs soffice
make test-integration           # build + run tests against the local binary
make test-integration-docker    # build + run inside the image
make test-all                   # samples + integration
```

`make fmt` runs `go fmt ./... && go mod tidy`. There is no `make
lint`, no `golangci-lint`, and no `go vet` target wired up â€” do not
introduce one without discussion. `make test` runs `go test -v ./...`
and will currently pass with no tests defined.

`make gen-ssl` produces a self-signed 4096-bit RSA cert (3650 days) in
`certs/` for local TLS smoke-tests.

`make clean` removes `bin/`, `test-server.log`, `/tmp/convert-req-*`,
`/tmp/{xlsb-to-xlsx,xlsx-to-ods,ods-to-xlsx}-req-*`, and
`/tmp/lo-profile-*`.

## Adding a new conversion direction

Example: add `ods-to-xlsb`.

1. `internal/server/convert.go` â€” extend the `toFormat != "xlsx" &&
   toFormat != "ods"` guard in `ConvertTo` to accept `"xlsb"`. If you
   want a typed call site, add a `ConvertXLSB` analog.
2. `internal/server/handlers.go`:
   - Add a `handleConvertOdsToXlsb` mirroring `handleConvertOdsToXlsx`,
     passing `"xlsb"` as the target format.
   - Update the `switch inputExt` arms in **both** `handleConvert` and
     `performJSONConversion` to include the new source extension and
     the new `.xlsb` target.
   - Add a new MIME to `contentTypeFor`.
3. `internal/server/routes.go` â€” register a new
   `e.POST("/api/v1/convert/ods-to-xlsb", handleConvertOdsToXlsb)`.
4. `api/openapi.yaml` â€” add the new path (mirror the existing
   `ods-to-xlsx` block) and update the description. Run
   `make generate` and commit the regenerated `internal/api/api.gen.go`.
5. `README.md` / `README.pt-BR.md` â€” add the row to the endpoints
   table.
6. `DOCUMENTS/scripts/test-api.sh` â€” add a multipart + JSON test
   mirroring the existing patterns. Add the new
   `ods-to-xlsb-req-` prefix to `Makefile`'s `clean` target.
7. If you want Swagger UI to advertise the new path, run with
   `--swagger` and verify `/docs`.

## Adding a new endpoint variant

Each typed handler in `internal/server/handlers.go` is a thin
dispatcher: branch on `isJSONContentType`, validate the extension, then
delegate to `performConversion` (multipart) or
`performJSONConversion` (JSON). Both helpers already handle the
`?format=json` / `Accept: application/json` override and the
upload/decode/cleanup pipeline. Copy an existing handler and change the
`toFormat` / `expectedExt` / `tempPrefix` arguments.

## What not to do

- Do not write Go unit tests against this codebase; the project
  intentionally tests through real HTTP + real LibreOffice. If a unit
  test seems necessary, raise it before adding it.
- Do not add new third-party packages casually. `go.mod` is intentionally
  small. New code should reach for the stdlib first
  (`slog`, `os`, `path/filepath`, `strings`, `encoding/base64`).
- Do not commit secrets, sample spreadsheets that contain PII, or large
  binaries. `testdata/` is git-ignored.
- Do not hand-edit `internal/api/api.gen.go` â€” it is generated. Edit
  `api/openapi.yaml` and run `make generate`.
- Do not replace the buffered-channel semaphore with a `sync.WaitGroup`
  or a `sync.Mutex`. The whole point is non-blocking acquire under
  context cancellation. Re-read the "Concurrency contract" section
  above.
- Do not split the inline `indexHTML` / `swaggerUIHTML` strings in
  `internal/server/templates.go` into a templating system or external
  assets. The pages are intentionally self-contained.
- Do not change `CGO_ENABLED=0` or remove `-trimpath` /
  `-ldflags="-s -w"` from the build recipe without thinking it through â€”
  the binary is meant to be portable across glibc/musl Debian variants
  and reproducible across CI / local builds.
- Do not introduce a linter or formatter config
  (`golangci-lint`, `.editorconfig`, `gofumpt` rules, etc.) without
  discussion. There is intentionally no linting step wired up today.
